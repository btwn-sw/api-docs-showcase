# Eventbrite API SDK

Eventbrite는 공식 SDK를 제공하지 않습니다. SDK 없이 표준 HTTP 클라이언트로
Eventbrite API를 바로 연동할 수 있습니다. 이 가이드는 사용 가능한 것이
무엇인지, 무엇을 피해야 하는지, 직접 연동을 어떻게 시작하는지
설명합니다.

## SDK 현황

Eventbrite는 어떤 프로그래밍 언어에 대해서도 공식 SDK를 유지 관리하지
않습니다. API는 표준 RESTful HTTP 인터페이스이며, 인증, 요청, 응답 처리를
직접 담당해야 합니다.

### 커뮤니티 SDK

Eventbrite API를 감싸는 커뮤니티 유지 라이브러리가 일부 있지만,
Eventbrite가 공식적으로 지원하지는 않습니다. 프로덕션에서 커뮤니티 SDK
사용을 고려한다면 아래를 확인하세요.

- 적용 범위가 불완전할 수 있습니다 — 모든 엔드포인트나 기능이 지원된다는
보장이 없습니다.
- API 변경 후 유지 관리가 늦어지면 조용한 오류가 발생할 수 있습니다.
- 코드와 API 응답 사이에 추상화 계층이 생겨서 디버깅이 어려워집니다.

프로덕션 연동에는 직접 HTTP 방식이 더 안전하고 예측 가능한 선택입니다.

<br>

## 직접 HTTP 연동

직접 HTTP 연동이 권장하는 방식입니다. 외부 의존성 없이 요청 구성,
오류 처리, 응답 파싱을 완전히 제어할 수 있습니다.

### 권장 도구

| 사용 환경 | 도구 |
| --- | --- |
| 브라우저 기반 JavaScript | `fetch` (내장) |
| Node.js / 서버 사이드 JavaScript | `node-fetch` 또는 `axios` |
| 테스트 및 탐색 | Postman |
| 기타 백엔드 언어 | 해당 언어의 표준 HTTP 라이브러리 |

cURL, JavaScript, Node.js 요청 예시는
[코드 예제]()를 참고하세요.

## 클라이언트 코드 생성

OpenAPI 명세가 있다면 코드 생성 도구로 해당 언어의 타입이 있는 HTTP
클라이언트를 만들 수 있습니다. 생성된 클라이언트는 반복 코드를 줄여주지만
프로덕션에서 사용하기 전에 검증과 커스터마이징이 필요합니다.

Eventbrite API OpenAPI 명세는 여기서 확인할 수 있습니다:
